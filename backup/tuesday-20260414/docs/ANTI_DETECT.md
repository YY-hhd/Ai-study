## ✅ 已启用的防检测功能

### 1️⃣ 随机延迟系统

每次操作后随机等待，模拟人工操作的不规则性。

```yaml
random_delay:
  min: 0.5    # 最小延迟 0.5 秒
  max: 2.0    # 最大延迟 2.0 秒
  enabled: true
```

**效果**: 避免固定节奏操作，降低被检测风险

---

### 2️⃣ 定时休息系统

每 5 分钟强制休息，避免长时间连续操作。

```yaml
break_interval: 300    # 每 300 秒 (5 分钟) 休息
break_duration: 60     # 每次休息 60 秒
break_reminder: true   # 显示休息提醒
```

**效果**: 
- 模拟真实用户的休息习惯
- 避免连续操作被标记
- 保护账号安全

**休息提醒示例**:
```
==================================================
😴 定时休息时间到！
⏰ 休息 60 秒，活动一下身体~
==================================================
```

---

### 3️⃣ 随机暂停（模拟用户查看）

10% 概率触发随机暂停，模拟用户查看屏幕/操作手机。

```yaml
random_pause:
  enabled: true
  probability: 0.1   # 10% 触发概率
  duration: [3, 10]  # 暂停 3-10 秒
```

**效果**: 增加操作真实性，降低自动化特征

---

### 4️⃣ 速度波动系统

操作速度随机波动±20%，模拟用户不同状态。

```yaml
speed_variation:
  enabled: true
  base_speed: 1.0
  variation: 0.2     # ±20% 波动
```

**效果**: 
- 避免固定速度模式
- 模拟用户疲劳/兴奋状态变化
- 每 30% 操作后重新计算速度因子

## ⚙️ 自定义配置

### 保守模式（推荐新手）

```yaml
safety:
  anti_detect: true
  random_delay:
    min: 1.0      # 更长延迟
    max: 3.0
  break_interval: 180    # 每 3 分钟休息
  break_duration: 120    # 休息 2 分钟
  random_pause:
    enabled: true
    probability: 0.2     # 20% 概率
    duration: [5, 15]    # 暂停 5-15 秒
  max_runtime: 3600      # 最大 1 小时
  max_loops_per_hour: 8  # 每小时最多 8 轮
```

### 激进模式（高风险）

```yaml
safety:
  anti_detect: false     # 关闭防检测
  max_runtime: 14400     # 最大 4 小时
  max_loops_per_hour: 20 # 每小时 20 轮
```

⚠️ **警告**: 激进模式封号风险极高！

## 📞 常见问题

**Q: 防检测模式会影响效率吗？**

A: 会有一定影响，但能显著降低封号风险。建议优先保证账号安全。

**Q: 可以关闭某些防检测功能吗？**

A: 可以，在 `config/settings.yaml` 中设置对应 `enabled: false`。

**Q: 如何查看当前防检测状态？**

A: 启动时会自动显示防检测配置，日志中也有详细记录。
