# WeGame 小号测试流程 - 安全验证指南

> ⚠️ **重要声明**: 本流程仅用于技术测试，使用外挂工具存在封号风险。请自行承担后果，**切勿用于主账号**。

---

## 📋 测试前准备

### 1️⃣ 账号准备

```
✅ 必须满足:
├─ 全新注册小号 (无充值、无珍贵道具)
├─ 与主号无关联 (不同手机/邮箱注册)
├─ 独立 IP 登录 (不用主号同一网络)
└─ 心理准备：随时可能封号

❌ 禁止使用:
├─ 主账号
├─ 充值过的账号
├─ 有珍贵道具/宠物的账号
└─ 与主号关联的账号
```

### 2️⃣ 环境准备

```bash
# 检查项目结构
cd ~/.openclaw/workspace/projects/roco-auto-farmer
tree -L 2

# 确认配置文件存在
ls -la config/settings_wegame.yaml

# 安装依赖 (如未安装)
pip install -r requirements.txt
```

### 3️⃣ WeGame 设置

| 设置项 | 推荐值 | 说明 |
|:---|:---:|:---|
| **游戏分辨率** | 1280×720 | 必须与配置一致 |
| **窗口模式** | 窗口化 | 非全屏 |
| **画质设置** | 中/低 | 减少识别误差 |
| **WeGame 覆盖** | 关闭 | F12 等快捷键关闭 |
| **游戏音效** | 可选 | 不影响识别 |

## 🧪 分阶段测试流程

### 阶段一：窗口识别测试 (5 分钟)

```bash
# 1. 启动 WeGame + 洛克王国世界
# 2. 运行窗口识别测试
python src/test_window_detect.py

# 预期输出:
# ✅ 找到窗口："洛克王国世界" (1280x720)
# ❌ 未找到窗口 → 调整窗口标题配置
```

**成功标准**:
- [ ] 正确识别窗口标题
- [ ] 分辨率匹配 1280×720
- [ ] 窗口坐标准确

**失败处理**:
```bash
# 用 Spy++ 查看真实窗口标题
# 或手动截图测试
python src/test_screenshot.py --manual
```

---

### 阶段二：图像识别测试 (10 分钟)

```bash
# 1. 截取游戏画面
python src/test_screenshot.py --game

# 2. 测试模板匹配
python src/test_template_match.py \
  --template templates/lava_mine/entry.png \
  --threshold 0.85

# 3. 验证关键元素识别
python src/test_recognition.py \
  --elements "energy,teleport,monster"
```

**成功标准**:
- [ ] 入口识别率 ≥90%
- [ ] 能量点识别准确
- [ ] 传送按钮识别准确
- [ ] 误识别率 ≤5%

**调整阈值**:
```yaml
# 如识别失败，临时降低阈值
recognition:
  threshold: 0.75  # 从 0.85 降至 0.75
```

---

### 阶段三：单轮手动测试 (15 分钟)

```bash
# 1. 单轮测试 (手动确认每步)
python src/main.py \
  --config config/settings_wegame.yaml \
  --map lava_mine \
  --loops 1 \
  --manual-confirm

# 2. 观察日志
tail -f logs/agent.log
```

**检查清单**:
- [ ] 窗口定位准确
- [ ] 寻路逻辑正确
- [ ] 采集动作执行
- [ ] 返回入口正常
- [ ] 无异常弹窗

**风险信号** (立即停止):
```
⚠️ 游戏弹窗警告
⚠️ 操作被拒绝/无效
⚠️ 识别率突然下降
⚠️ 日志出现异常错误
```

---

### 阶段四：短时自动测试 (30 分钟)

```bash
# 1. 运行 3 轮自动测试
python src/main.py \
  --config config/settings_wegame.yaml \
  --map lava_mine \
  --loops 3 \
  --timeout 1800  # 30 分钟超时

# 2. 监控系统资源
watch -n 5 'ps aux | grep python | head -3'
```

**监控指标**:
| 指标 | 正常值 | 异常处理 |
|:---|:---:|:---|
| CPU 占用 | ≤25% | 降低运行速度 |
| 内存占用 | ≤500MB | 检查内存泄漏 |
| 识别成功率 | ≥85% | 调整阈值 |
| 单轮耗时 | 3-5 分钟 | 检查卡点 |

**日志分析**:
```bash
# 查看错误统计
grep -i "error\|fail\|timeout" logs/agent.log | wc -l

# 查看识别成功率
grep "recognition" logs/agent.log | tail -20
```

### 阶段五：长时稳定性测试 (2 小时)

```bash

python src/main.py \
  --config config/settings_wegame.yaml \
  --map lava_mine \

  --map lava_mine \

ps aux | grep "main.py"
```

**成功标准**:
- [ ] 完成 10 轮无崩溃
- [ ] 平均成功率 ≥80%
- [ ] 无游戏警告/弹窗
- [ ] 资源占用稳定

## 🚨 紧急情况处理

### 检测到异常

```bash
# 1. 立即停止 (热键 F12 或手动)
pkill -f "main.py"

# 2. 保存日志
cp logs/agent.log logs/emergency_$(date +%Y%m%d_%H%M%S).log

# 3. 截图保存
python src/test_screenshot.py --emergency

# 4. 退出游戏
# 正常退出，不要强制关闭
```

### 账号被封

```
处理流程:
1. 记录封号时间、原因、时长
2. 保存相关日志和截图
3. 分析可能原因 (见下方)
4. 决定是否申诉 (小号通常不申诉)
5. 总结教训，调整配置
```

### 封号原因分析

| 原因 | 特征 | 改进方案 |
|:---|:---|:---|
| **操作过快** | "异常操作" | 增加延迟，降低轮次 |
| **长时间运行** | "挂机行为" | 缩短时长，增加休息 |
| **固定节奏** | "脚本特征" | 增强随机化 |
| **多开关联** | "批量操作" | 单开，独立 IP |
| **举报** | "玩家举报" | 更隐蔽，避开人流 |
