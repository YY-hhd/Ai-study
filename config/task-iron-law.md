# 任务铁律 - Task Iron Law

## 执行流程

```
1. 分解思考任务步骤
2. 开始执行
3. 遇到问题 → 改变方法再尝试
4. 至少尝试 5 轮后再找用户求助
```

## 停止条件

| 条件 | 触发值 | 说明 |
|:---|:---:|:---|
| **尝试轮次** | 5 轮 | 标准尝试次数 |
| **单任务 Token** | 8,000 | 百炼 Lite 套餐限制 |
| **每日预算** | 50,000 | 超 80% 强制高速池 |
| **需要授权** | 支付/授权操作 | 需用户确认 |
| **安全风险** | 删除/系统配置 | 需用户确认 |

## 尝试策略

| 轮次 | 策略 |
|:---|:---|
| 第 1 轮 | 直接执行 - 使用最直接的方法 |
| 第 2 轮 | 换个方法 - 尝试不同的实现方式 |
| 第 3 轮 | 查阅文档 - 搜索官方文档和教程 |
| 第 4 轮 | 搜索解决方案 - web_search 查找类似问题 |
| 第 5 轮 | 组合多种方法 - 综合前面所有尝试 |

## 安全边界（需确认）

- ❌ 删除操作（`rm -rf`、`drop table`、永久删除文件）
- ❌ 系统配置修改（/etc/ 目录、系统服务）
- ❌ 外部发送（邮件、推文、公开消息）
- ❌ 支付操作（任何涉及金钱的交易）
- ❌ 权限变更（chmod 777、sudo 提权）

## 错误处理原则

1. **记录错误**: 详细记录错误信息和上下文
2. **分析原因**: 理解为什么会失败
3. **尝试变通**: 寻找替代方案
4. **总结经验**: 将解决方案记录到 MEMORY.md

## Token 管理

- **单任务上限**: 20,000 tokens
- **压缩触发**: 5,000 tokens 或 10 轮对话
- **压缩目标**: 节省 22% tokens

## 示例

```
用户：帮我安装一个 Python 包

AI 执行:
[第 1 轮] pip install package-name → 失败（权限问题）
[第 2 轮] pip install --user package-name → 成功
[记录] 将解决方案记录到 TOOLS.md
```
