## 🧠 可固化技能

### 技能 1: WSL2 浏览器配置方案 🆕

**适用场景**: WSL2 环境下 OpenClaw 无法自动检测 Windows 浏览器

**配置步骤**:
```bash
# 1. 配置 browser.executablePath
openclaw config set browser.executablePath "/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"

# 2. 重启网关
openclaw gateway restart
```

**好处**:
- ✅ 完整 browser 工具支持
- ✅ 网页自动化能力解锁
- ✅ 统一接口，无需区分 exec 和 browser

---

### 技能 2: Remote CDP 跨系统浏览器控制 🆕🔥

**适用场景**: WSL2 需要控制 Windows Chrome 实现完整浏览器自动化

**问题诊断流程**:
```
1. browser 工具报错 "timed out"
2. 检查 CDP 连接：curl http://172.20.240.1:9222/json/version
3. 检查端口转发：netsh interface portproxy show all
4. 检查防火墙：Windows Defender 防火墙设置
```

**配置步骤**:

**Windows PowerShell (管理员)**:
```powershell
# 1. 添加端口转发
netsh interface portproxy add v4tov4 listenport=9222 listenaddress=0.0.0.0 connectport=9222 connectaddress=127.0.0.1

# 2. 添加防火墙规则
New-NetFirewallRule -DisplayName "Chrome CDP 9222" -Direction Inbound -Protocol TCP -LocalPort 9222 -Action Allow
```

**启动 Chrome**:
```cmd
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222
```

**验证连接**:
```bash
# WSL2 中运行
curl http://172.20.240.1:9222/json/version
```

**好处**:
- ✅ 完整浏览器自动化 (点击/输入/截图/PDF)
- ✅ 可抓取 JS 渲染的动态内容
- ✅ 可登录网站后操作
- ✅ 支持批量任务自动化

**通用路径参考**:
| 浏览器 | WSL2 路径 |
|:---|:---|
| Chrome | `/mnt/c/Program Files/Google/Chrome/Application/chrome.exe` |
| Brave | `/mnt/c/Program Files/BraveSoftware/Brave-Browser/Application/brave.exe` |
| Edge | `/mnt/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe` |
